# 語言規範

## 主要語言

本專案使用**繁體中文（台灣用語）**作為主要語言。

## 適用範圍

### 必須使用繁體中文的部分

1. **使用者介面**
   - 所有按鈕文字
   - 遊戲說明
   - 訊息提示
   - 標籤與標題

2. **文件**
   - README.md
   - 規格文件（requirements.md、design.md、tasks.md）
   - Steering 文件（本目錄下所有 .md 檔案）
   - 程式碼註解

3. **程式碼註解**
   - JSDoc 註解
   - 內嵌註解
   - 說明性註解

### 使用英文的部分

1. **程式碼識別符號**
   - 類別名稱（例如：`GameField`、`TetrominoManager`）
   - 方法名稱（例如：`moveDown()`、`checkCollision()`）
   - 變數名稱（例如：`currentTetromino`、`lastDropTime`）
   - 常數名稱（例如：`FIELD_WIDTH`、`SCORE_RULES`）

2. **技術術語**
   - HTML、CSS、JavaScript
   - Canvas API
   - Git commit 訊息可使用英文

## 台灣用語慣例

使用台灣常用的技術術語：

- 使用「檔案」而非「文件」（指 file）
- 使用「程式碼」而非「代碼」
- 使用「函式」而非「函數」
- 使用「類別」而非「類」
- 使用「方法」而非「方法」
- 使用「變數」而非「變量」
- 使用「陣列」而非「數組」
- 使用「字串」而非「字符串」
- 使用「瀏覽器」而非「瀏覽器」
- 使用「伺服器」而非「服務器」
- 使用「網路」而非「網絡」

## AI 助手指引

當 AI 助手在此專案中工作時：

1. **回應語言**：使用繁體中文（台灣用語）回應使用者
2. **文件撰寫**：所有新建或修改的文件都應使用繁體中文
3. **程式碼註解**：新增的註解應使用繁體中文
4. **保持一致性**：遵循現有程式碼的註解風格與用語習慣
5. **程式碼本身**：類別、方法、變數名稱維持英文，遵循既有命名慣例
